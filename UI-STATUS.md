# UI 重做 —— 第 1 版(圆形数据处理流程,核心视觉)

按日志里那次"大改 UI"的方向(严肃科研工具风 + 六瓣花圆形流程图),并适配新架构
(组学模块 = 数据处理,无上下游之分)做了第一版。

## 这一版做了什么
- **设计系统刷新**(globals.css + tailwind.config.js):偏冷的科研灰白配色、Inter 字体栈、
  卡片/弹层阴影 token、步骤 5 态配色(未配置/已配置/运行中/完成/失败)、圆形流程动效。
- **圆形数据处理流程图**(新组件 `src/components/PipelineFlow.tsx`):
  六个步骤围成一圈 —— SRA/fastq → 质控 → 过滤 → 比对(STAR) → 定量 → 标准化,
  圆心是「一键运行」(带完成度进度环),点步骤进该步参数,运行中节点有呼吸光环。
- **接进转录组数据处理页**(`src/routes/Upstream.tsx`):用圆形流程替换原来扁平的标签条,
  圆心按钮直接跑一键流程(到标准化,线程用项目级计算资源);
  "STAR 索引 / 导入 counts"放到下面作为次要入口。

## 现在的状态显示
- **已配置/未配置**:根据项目里保存过的参数推导(配过参数的步骤亮蓝)。
- **运行中/完成/失败**:还没接实时任务状态 —— 这是下一步(把 jobs 状态喂给流程图)。
