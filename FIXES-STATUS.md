# 本轮修复

## ① 不用每次手动 pin time —— 已修
core/src-tauri/Cargo.toml 里加了 `time = "=0.3.47"`,cargo 不会再解析到坏的 0.3.48。
以后正常 `bash build-and-install.sh --skip-env` 即可,不用再 cargo update。
(只需在用本版第一次构建前清一次旧产物:`rm -rf core/src-tauri/target`,因为之前用 0.3.48 编过的缓存要清掉。之后就不用了。)

## ② analysis 模块不再需要单独构建 —— 已修
build-and-install.sh 现在:某个模块若还没建过 conda env(比如新加的 analysis),
即使你传了 --skip-env,也会自动为它建一次 env 再打包;已有 env 的模块仍然复用、不重建。
所以一条 `bash build-and-install.sh --skip-env` 会把转录组 + 分析两个模块都装上。

前端按钮、确认框、密码框、后端 apt remove、包名映射我都查了,接线是通的
(omics-analysis → plantomics-module-analysis,正确)。"没反应"通常是两种情况之一:
  (a) 卸载其实成功了,但模块列表是启动时加载进内存的,**重启前不会消失** → 看起来"没反应";
  (b) apt remove / sudo 密码那步失败了。
